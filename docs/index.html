PANIC

Our Story

As college students, time is always in short supply. Traditional programming languages require complex syntax and deep familiarity with detailed documentation, which can slow you down when you’re racing against a deadline. The PANIC programming language was designed with this in mind, offering a streamlined, efficient approach that cuts the complexity so you can write, debug, and run your programs quickly, even if you only have a quick minute to get it done.


PANIC achieves this efficiency by blending shorthand abbreviations with the option to use full descriptive commands. This approach allows users to choose a more concise, rapid coding style when speed is essential, or a more traditional format when clarity and readability are more important. PANIC's flexible keywords and simple notation reduce the risk of small syntax errors preventing your program from running smoothy the first time. This also makes debugging faster, more intuitive and much easier to understand.


Ultimately, PANIC is more than just another coding language, it’s a practical tool for modern software challenges. Whether you’re tackling an urgent assignment or experimenting with new ideas, PANIC provides a flexible, high-performance environment that allows you to focus on scripting efficiently rather than fighting with a language's verbose syntax. This makes it an ideal choice for both the procrastinating student and the experienced developer seeking a fresh approach to development.

Key Features

  • Targeted for Learners & Hardcore Programmers

    Minimal syntactic sugar to let you focus on scripting efficiently

  • Shorthand Syntax

    Single-letter abbreviations (e.g. f for function, r for return) for quick scripting

  • Fast Scripting and Debugging

    Designed for rapid development with easy-to-use shortcuts

  • Python-Inspired Indentation

    Uses indentation instead of brackets, with multiple keyword spellings (e.g. p and print)

  • Streamlined Loop Ranges

    Uses the ... operator for loop ranges, inspired by Swift

  • Strong Static Typing

    Employs a robust type system similar to Swift’s

  • Flexible Function Calls

    Supports keyword and positional arguments as well as parameters, similar to Python

  • Functional Programming Features

    First-class functions with built-in support for function composition and pipelining using left/right arrow notation

Example Programs

Example 0: Hello World

PANIC JS 
# Print a message
p "Hello, World!" 
console.log("Hello, World!");

Starting out simple, printing is very easy in PANIC, simply write print or p. Furthermore, one can either use parenthesis or not for function calls.

Example 1: Comparision function

PANIC JS 
f compare_numbers(a, b):
  if a = b:
    p a, " = ", b
  elif a < b:
    p a, " < ", b
  else:
    p a, " > ", b

# Variable Assignment
n1: 10 
n2: 20

# Function Call
compare_numbers(n1, n2)  
function compareNumbers(a, b) {
  if (a === b) {
    console.log(a + " = " + b);
  } else if (a < b) {
    console.log(a + " < " + b);
  } else {
    console.log(a + " > " + b);
  }
}

let n1 = 10;
let n2 = 20;

compareNumbers(n1, n2);

In PANIC function declarations need to have explicitly typed parameters. In addition, if, else, and elif statements are very Pythonic, however, they don't need to end with colons. Variable assignments follow the form name: literal. This is because we use the equals sign to test for equality. Because the function returns void the return type does not need to explicitly be marked, similar to Swift.

Example 2: Fibonacci

PANIC JS 
f fibonacci(n):
  # Initialize starting values
  a: 0  
  b: 1

  # Loop from 0 to n
  l i in [0...n]:  
    p a
    next: a + b
    a: b
    b: next

num_terms: 10

fibonacci(num_terms)
 
function fibonacci(n) {
  let a = 0,
      b = 1;

  for (let i = 0; i < n; i++) {
    console.log(a);
    let next = a + b;
    a = b;
    b = next;
  }
}

let numTerms = 10;

fibonacci(numTerms);

In PANIC l stands for loop and it is used both for whiles and for loops. We use the ... for shorthand notation to make ranges. Ranges start from the first number and include the last number, but don't go above it.

Example 3: Factorials with tail recursion

PANIC JS 
f factorial(n):
  if n = 0:
    r 1
  elif n = 1:
    r 1
  r n*factorial(n-1)

  p factorial(5)
 
function factorial(n) {
  if (n === 0 || n === 1) {
    return 1;
  }
  return n * factorial(n - 1);
}

console.log(factorial(5));

Here we have to declare a return type for the function factorial. Also, or is spelled out instead of being ||.

Example 4: Recursive looping

PANIC JS 
l i in [1...5]:
  l j in i:
    p "*"
      p " "
 
for (let i = 1; i <= 5; i++) {
  let row = "";
  for (let j = 1; j <= i; j++) {
    row += "* ";
  }
  console.log(row);
}

In PANIC arguments to a function can either be positional or keyword. Positional arguments come first, then keyword args, and ... is used to collect the rest of the arguments into an array.

Example 5: Anonymous Functions, Range and Maps

PANIC JS 
d int: int x r x + 1
b: [1...7, +2]
b -> d -> p     
p <- d <- b
 
d = (x) => x + 1;
b = [1, 3, 5, 7];
x = b.map(d);
console.log(x);

This is an example of functional programming and first-class functions in our language. In our language, the arrows show the flow of data, which can be written both ways. Despite the way it is written the function composition will still be the same: in this case p(d(b)).
