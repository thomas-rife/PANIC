PANIC

Our Story

As college students, time is always in short supply. Traditional programming languages require complex syntax and deep familiarity with detailed documentation, which can slow you down when you’re racing against a deadline. The PANIC programming language was designed with this in mind, offering a streamlined, efficient approach that cuts the complexity so you can write, debug, and run your programs quickly, even if you only have a quick minute to get it done.


PANIC achieves this efficiency by blending shorthand abbreviations with the option to use full descriptive commands. This approach allows users to choose a more concise, rapid coding style when speed is essential, or a more traditional format when clarity and readability are more important. PANIC's flexible keywords and simple notation reduce the risk of small syntax errors preventing your program from running smoothy the first time. This also makes debugging faster, more intuitive and much easier to understand.


Ultimately, PANIC is more than just another coding language, it’s a practical tool for modern software challenges. Whether you’re tackling an urgent assignment or experimenting with new ideas, PANIC provides a flexible, high-performance environment that allows you to focus on scripting efficiently rather than fighting with a language's verbose syntax. This makes it an ideal choice for both the procrastinating student and the experienced developer seeking a fresh approach to development.

Key Features

  • Targeted for Learners & Hardcore Programmers

    Minimal syntactic sugar to let you focus on scripting efficiently

  • Shorthand Syntax

    Single-letter abbreviations (e.g. f for function, r for return) for quick scripting

  • Fast Scripting and Debugging

    Designed for rapid development with easy-to-use shortcuts

  • Streamlined Loop Ranges

    Uses the ... operator for loop ranges, inspired by Swift

  • Strong Static Typing

    Employs a robust type system similar to Swift’s

  • Flexible Function Calls

    Supports keyword and positional arguments as well as parameters, similar to Python

Example Programs

Example 0: Hello World

PANIC JS 
# Print a message
p("Hello, World!") 
console.log("Hello, World!");

Starting out simple, printing is very easy in PANIC. Simply write print or p, and you can use parentheses or not for function calls.

Example 1: Comparison Function

PANIC JS 
# Function Definition 
f compare_numbers(x int y int) {  
  if x = y {
    p(x " is equal to " y)
  }
  elif x < y {
    p(x " is less than " y)
  }
  else {
    p(x " is greater than " y)
  }
}

# Variable Assignment
im n1: 10 
im n2: 20

# Call the function
compare_numbers(n1 n2)
 
function compareNumbers(a, b) {
  if (a === b) {
    console.log(a + " is equal to " + b);
  } else if (a < b) {
    console.log(a + " is less than " + b);
  } else {
    console.log(a + " is greater than " + b);
  }
}

let n1 = 10;
let n2 = 20;

compareNumbers(n1, n2);

In PANIC, function declarations have explicitly typed parameters. The conditional statements (if, elif, else) resemble Python but without requiring colons, and variable assignments use the form name: literal (since the equals sign is used for equality tests).

Example 2: Fibonacci

PANIC JS 
f fibonacci(n int) {
  # Initialize starting values
  mu x: 0 
  mu y: 1

  # Loop from 0 to n 
  l i in [0...n] {  
    p(x " ")
    mu z: x
    x : y
    y: z + x
  }

  # Use "r" for return
  r 
}

im num_terms: 10
fibonacci(num_terms)
 
function fibonacci(n) {
  let a = 0,
      b = 1;
  for (let i = 0; i < n; i++) {
    console.log(a);
    let next = a + b;
    a = b;
    b = next;
  }
}

let numTerms = 10;
fibonacci(numTerms);

In PANIC, l stands for loop, which is used for both for and while loops. Mutable variables are denoted as mu and immutable ones are denoted as im. Ranges are written using ..., which includes the starting and ending numbers.

Example 3: Factorials with Tail Recursion

PANIC JS 
f factorial(n int) -> int {
  if n = 0 or n = 1 {
    r 1 
  }
  r n * factorial(n-1)
}
p(factorial(5))
 
function factorial(n) {
  if (n === 0 || n === 1) {
    return 1;
  }
  return n * factorial(n - 1);
}

console.log(factorial(5));

In PANIC, you must declare a return type for the function. Also, logical operators like or are spelled out instead of using ||.

Example 4: Recursive Looping

PANIC JS 
l i in [1...5] {
  l j in i {
    p("*")
  }
  p(" ")
}

for (let i = 1; i <= 5; i++) {
  let row = "";
  for (let j = 1; j <= i; j++) {
    row += "* ";
  }
  console.log(row);
}

In PANIC, function arguments can be either positional or keyword-based. Positional arguments come first, then keyword arguments, and ... is used to capture additional arguments into an array.

Example 5: Arrays

PANIC JS 

# Create immutable array ranging from 1 to 20, each multiplied by 2
im x: [1...20, *2]

# Create a nested array structure
im y: [[[x]]]

# Access the first element of the nested array
mu z: y[0][0][0]

# Modify the value of z to a new array
z: [1 2 3 4 5]
              

const x = Array.from({ length: 20 }, (_, i) => (i + 1) * 2);

const y = [[[x]]];

let z = y[0][0][0];

z = [1, 2, 3, 4, 5];

This is how arrays are done in PANIC compared to JavaScript. The arrays all have types and they are all checked.

Example 6: Classes

PANIC JS 
# Define a class Dog
c Dog {
  # Define constructor
  con(name string)
    f bark(sound string) -> string {
        im greet: name + " says " + sound
        return greet
    }
}

# Create a new Dog
im dog: Dog("rocky")

dog.bark("woof")
 
class Dog {
  constructor(name) {
    this.name = name;
  }
  
  bark(sound) {
    const greet = this.name + " says " + sound;
    return greet;
  }
}

const dog = new Dog("rocky");
console.log(dog.bark("woof"));

This is how classes are done in PANIC compared to JavaScript. Classes are defined using cc with constructors being defined using con. Objects can be created as immutable or mutable.
